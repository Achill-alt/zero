## 3. Bug 修复记录

### Bug #1: 审批提交返回 500

- **症状**：`POST /contracts/{id}/submit` 返回 500 Internal Server Error，但审批实例实际已创建
- **原因**：`contract_service.py:submit_contract()` 返回的 `result` dict 包含 `instance`（SQLAlchemy ORM 对象），FastAPI JSON 序列化失败
- **修复**：将 `result` dict 中的 `instance` 替换为可序列化的 `instance_id`、`status`、`current_step_index` 字段
- **文件**：[backend/app/services/contract_service.py:87-92](backend/app/services/contract_service.py)

### Bug #2: FTS5 全文搜索 500 错误

- **症状**：`GET /search?q=Test` 返回 `sqlite3.InterfaceError: column index out of range`
- **原因**：FTS5 `snippet()` 函数的列索引是 **0-based**（0=title, 1=content），代码错误使用了 1-based 索引（1=title, 2=content），访问了不存在的列索引 2
- **修复**：
  - `snippet(contracts_fts, 1, ...)` → `snippet(contracts_fts, 0, ...)` (title)
  - `snippet(contracts_fts, 2, ...)` → `snippet(contracts_fts, 1, ...)` (content)
- **文件**：[backend/app/services/search_service.py:15-16](backend/app/services/search_service.py)

### Bug #3: FTS5 搜索结果高亮逻辑错误

- **症状**：`not q` 条件导致有搜索词时反而不返回高亮片段
- **修复**：`hasattr(row, 'title_highlight') and not q` → `q`（有搜索词时返回高亮）
- **文件**：[backend/app/services/search_service.py:75-76](backend/app/services/search_service.py)
