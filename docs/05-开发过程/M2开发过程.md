## 3. M2 新增修复

### 修复 #1：新建 Search.vue — 全文检索页面

- **问题**：FTS5 搜索 API 在 M1 已验证通过，但前端缺少搜索页面，用户无法从 UI 使用全文检索
- **修复**：创建 [frontend/src/views/Search.vue](frontend/src/views/Search.vue)
  - 搜索输入框 + 类型/状态筛选
  - FTS5 搜索结果展示（含 `<mark>` 高亮片段渲染）
  - 分页支持
  - 点击结果卡片跳转到合同详情
- **路由**：`/search` → `Search.vue`
- **导航**：侧边栏新增 "全文检索" 菜单项（Search 图标）

### 修复 #2：Dashboard 统计数据不准确

- **问题**：Dashboard 只取前 5 条合同，`审批中` 和 `已归档` 计数从 5 条中客户端过滤，不能反映真实数据
- **修复**：[frontend/src/views/Dashboard.vue](frontend/src/views/Dashboard.vue)
  - 改为 4 个并行 API 调用：
    - `list({page_size:5})` → 最近合同 + 总数
    - `list({status:'pending_approval', page_size:1})` → 审批中准确总数
    - `list({status:'archived', page_size:1})` → 已归档准确总数
    - `expiring({days:30})` → 即将到期总数
  - 每个统计卡片的值来自对应 API 的 `total` 字段

### 修复 #3：ContractDetail 审批权限校验过宽

- **问题**：`canApprove` 计算属性对任何 approver 角色都返回 `true`，导致不匹配当前审批步骤的审批人也能看到"通过/驳回"按钮
- **修复**：[frontend/src/views/ContractDetail.vue](frontend/src/views/ContractDetail.vue)
  - 页面加载时额外调用 `GET /approvals/pending` 获取当前用户的待审批列表
  - `canApprove` 改为：检查当前合同 ID 是否出现在用户待审批列表中
  - 后端有 403 兜底，前端按钮状态现在准确反映实际权限

### 修复 #4：新增撤回审批前端交互

- **问题**：后端 `POST /approvals/{id}/withdraw` API 和测试均已就绪，但前端缺少撤回按钮
- **修复**：[frontend/src/views/ContractDetail.vue](frontend/src/views/ContractDetail.vue)
  - 新增 `canWithdraw` 计算属性：合同状态为 `pending_approval` + 当前用户是合同创建人 + 存在进行中的审批实例
  - 新增 "撤回审批" 按钮（warning 类型）
  - 新增 `handleWithdraw` 函数：确认弹窗 → 调用 API → 刷新页面状态
