1. **合同全生命周期**：创建 → 编辑 → 提交 → 四级审批（dept_manager→legal→finance_director→ceo） → 审批通过 → 归档 ✅

### 3.4 过程中发现并修复的问题

| # | 问题 | 类型 | 修复 |
|---|------|------|------|
| 1 | E2E 脚本中 API 响应格式解析不一致（`data` 包装层） | 脚本问题 | 统一使用 `d(r)` 辅助函数提取 data |
| 2 | approve API 的 `message` 字段在顶层而非 data 内 | 脚本适配 | 使用 `r.json()["message"]` |
| 3 | 模板列表路径为 `/contracts/templates/all` 而非 `/contracts/templates` | 脚本适配 | 修正 URL 路径 |
| 4 | 作废合同提交返回 409 而非 400 | 脚本适配 | 改为 `>= 400` 检查 |

> 以上均为 E2E 脚本的格式适配问题，系统本身无缺陷。
