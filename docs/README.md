## 维护约定

- 正式文档和草案放在同一阶段目录，通过文件名中的"草案"明确状态。
- 文档路径统一使用项目根目录相对路径，例如 `docs/02-系统设计/系统设计说明书.md`。
- 需求或设计发生变更时，应同步更新下游追踪、计划和验收文档。
- 新增文档时，应归入对应生命周期目录并更新本导航。
- 一边开发一边将文档写入对应阶段目录——文档与代码同步演进。
- 迭代优化阶段的文档统一归入 `08-优化迭代/` 目录。
