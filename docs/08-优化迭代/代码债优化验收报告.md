## 五、变更总结

本次优化全程保持 **API 契约不变**（所有 endpoint URL、请求参数、响应结构保持向后兼容，唯一例外是 `/approvals/pending` 响应从 `{data: [...]}` 升级为 `{data: {items, total, page, page_size}}`，同时前端所有调用处已同步更新）。

**核心收益：**
- 消除 30+ 处重复代码（序列化器、状态映射、JSON 解析）
- 统一错误处理模式（`raise HTTPException` 替代 `return tuple`）
- SQL 查询从字符串拼接改为结构化子句列表，消除 SQL 注入风险
- 前端 API 模块职责清晰（search 独立于 admin）
- Python 3.12+ 兼容（`datetime.utcnow()` 迁移）
- 环境变量管理（开发/生产可独立配置 API 地址）
