# 用户注册校验错误中文化

> **日期：** 2026-07-10
> **版本：** v1.2
> **状态：** ✅ 已完成

## 问题

管理员在用户管理页面新增用户时：
1. 校验失败只显示一个 × 图标，没有文字提示
2. 错误信息是英文的（如 `String should have at least 3 characters`）
3. 无法判断具体是哪个字段出了什么问题

## 根因

1. **后端：** FastAPI/Pydantic 的 `RequestValidationError` 返回 422 时，`detail` 为英文字段数组格式 `[{loc: [...], msg: "String should have at least 3 characters", type: "string_too_short"}]`
2. **前端：** 错误拦截器将数组直接传给 `ElMessage.error()`，数组转字符串后显示 `[object Object]`，用户只看到 ×

## 修复

### 后端：自定义校验异常处理器

在 `backend/app/main.py` 注册 `RequestValidationError` 处理器，将 Pydantic 错误翻译为中文：

| Pydantic 错误类型 | 中文翻译 |
|-------------------|----------|
| `missing` | `「字段名」为必填项` |
| `string_type` | `「字段名」必须是文本` |
| `integer_type` | `「字段名」必须是整数` |
| `string_too_short` | `「字段名」长度不足，至少需要 N 个字符` |
| `string_too_long` | `「字段名」超出长度限制，最多 N 个字符` |
| `value_error.any_str.min_length` | 同上（Field(min_length=...) 用 ctx.limit_value） |
| `value_error.any_str.max_length` | 同上（Field(max_length=...) 用 ctx.limit_value） |

### 前端：错误拦截器兼容数组

`frontend/src/api/index.ts` 错误拦截器增加数组检测：

```typescript
if (Array.isArray(detail)) {
  msg = detail.map((e) => {
    const field = e.loc?.slice(-1)[0] || ''
    return field ? `${field}: ${e.msg}` : e.msg
  }).join('；')
} else {
  msg = detail || error.message || '请求失败'
}
```

### 前端：表单客户端校验

`UserManage.vue` `handleRegister()` 函数：
- 新增 try/catch 防止未捕获异常
- 提交前客户端校验：用户名 ≥ 3 字符、密码 ≥ 6 字符
- 不满足条件时 `ElMessage.warning` 中文提示

| `frontend/src/api/index.ts:26-35` | 错误拦截器处理数组 detail |

## 验收

1. 管理员登录 → 用户管理 → 新增用户
2. 用户名为空 → 提示「`「username」长度不足，至少需要 3 个字符`」
3. 密码不足 6 位 → 提示「`「password」长度不足，至少需要 6 个字符`」
4. 正常填写 → 注册成功
