# 代码债优化方案

- 文档版本：v1.0
- 文档状态：已确认
- 编制日期：2026-07-08
- 依据：项目复盘 v1.0 + 全代码审查

## 1. 背景与目标

MVP v1.0 在 2 天 vibecoding 内完成了 57 项开发任务，但追求交付速度的同时积累了一些代码债。本次优化以 **"先治代码债，再谈新功能"** 为策略，在不改变 API 契约和用户可见行为的前提下，提升代码质量、可维护性和一致性。

**范围内：**
- 消除重复代码（序列化、JSON 解析、前端工具函数）
- 统一错误处理模式
- SQL 查询性能优化（搜索服务、到期预警）
- 前端共享逻辑抽取（composables）
- 前端错误处理改善
- API 模块整理
- 废弃 API 迁移

**范围外（留待 v1.1 功能迭代）：**
- jieba 中文分词搜索
- Docker 容器化部署
- Windows BAT 一键启动脚本
- 合同附件上传
- 邮件通知
- TypeScript 迁移

### 3.5 OPT-005 到期预警 SQL 化

**涉及文件：** `backend/app/services/contract_service.py:132-152`

**当前问题：**
```python
all_contracts = db.query(Contract).filter(...).all()  # 全量加载
for c in all_contracts:
    days_left = (c.end_date - today).days  # Python 内存循环
    if days_left <= days:
        expiring.append(...)
```

**优化方案：**
```python
from sqlalchemy import func

def get_expiring_contracts(db, days=30, page=1, page_size=20):
    today = date.today()
    cutoff = today + timedelta(days=days)

    query = (
        db.query(Contract)
        .filter(
            Contract.status.in_(["approved", "archived", "pending_approval"]),
            Contract.end_date <= cutoff,
        )
        .order_by(Contract.end_date.asc())
    )

    total = query.count()
    items = query.offset((page - 1) * page_size).limit(page_size).all()

    result_items = []
    for c in items:
        days_left = (c.end_date - today).days
        result_items.append({
            "contract": c,
            "days_left": days_left,
            "expired": days_left < 0,
        })

    return {"items": result_items, "total": total, "page": page, "page_size": page_size}
```

```javascript
export function useStatus() {
  const statusTagMap = {
    draft: 'info', pending_approval: 'warning', approved: 'success',
    archived: '', voided: 'danger',
  }
  const statusTextMap = {
    draft: '草稿', pending_approval: '审批中', approved: '已通过',
    archived: '已归档', voided: '已作废',
  }
  const typeTextMap = {
    purchase: '采购', sales: '销售', service: '服务',
    lease: '租赁', other: '其他',
  }

  function statusTag(s) { return statusTagMap[s] || 'info' }
  function statusText(s) { return statusTextMap[s] || s }
  function typeText(t) { return typeTextMap[t] || t }

  return { statusTag, statusText, typeText }
}
```

### 3.7 OPT-007 前端错误处理改善

**涉及文件：** 全部 View 组件（14 个）

**当前模式：**
```javascript
try { ... } catch { /* ignore */ }
```

**优化方案：**

由于 [api/index.js](frontend/src/api/index.js) 的 response interceptor 已统一调用 `ElMessage.error(msg)`，View 层的 `catch {}` 是冗余且有害的——它吞掉了 interceptor 已经弹出的错误后的 Promise rejection，反而可能导致调用方以为成功。

- **通用情况：** 移除 `catch {}`，让错误自然传播给 interceptor 处理
- **需要自定义处理：** 保留 catch 并写明确的错误逻辑（如撤回找不到实例时 `ElMessage.warning(...)`）

**需修改的具体位置：**
- `Dashboard.vue` line 79: `catch { /* ignore */ }` → 移除
- `ContractList.vue` line 75: `catch { /* ignore */ }` → 移除
- `ContractList.vue` line 107: `catch { /* ignore */ }` → 保留（有自定义 toast）
- `Search.vue` line 73: `catch { /* ignore */ }` → 移除
- 其余 View 逐一检查
