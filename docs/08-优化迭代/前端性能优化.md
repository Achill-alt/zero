# 前端性能优化：Element Plus 按需加载

> **日期：** 2026-07-09  
> **版本：** v1.2  
> **状态：** ✅ 已完成

## 问题

开发模式下前端界面加载和页面切换非常慢，甚至显示超时。具体表现：

- 首次加载页面需等待 8-10 秒
- 页面切换需要 3-5 秒
- 有时直接超时

## 根因分析

`main.ts` 中的 `app.use(ElementPlus)` 全量导入了 Element Plus 的全部 200+ 个组件：

```typescript
// 之前：全量导入 ~200 个组件
import ElementPlus from 'element-plus'
app.use(ElementPlus, { locale: zhCn })
```

**影响**：

| 阶段 | 影响 |
|------|------|
| Vite 预构建 | 需要处理 element-plus 入口文件及其引用的~1000 个子模块 |
| 浏览器首次加载 | 加载并执行全部 ~200 个组件的 JS 代码 |
| 页面切换 | Vue 在每次路由变化时检查所有 ~200 个全局组件 |
| 生产构建 | vendor-element chunk 达 1,015 KB |

而实际项目中仅使用了 **31 个** Element Plus 组件（`<el-button>`, `<el-table>`, `<el-form>` 等）。

## 解决方案

### 按需注册组件

创建 `frontend/src/element-plus.ts`，仅导入实际使用的 31 个组件：

```typescript
import {
  ElRow, ElCol, ElButton, ElBadge, ElTag, ElIcon,
  ElForm, ElFormItem, ElInput, ElInputNumber, ElSelect, ElOption,
  ElDatePicker, ElSwitch, ElUpload,
  ElTable, ElTableColumn, ElPagination, ElDescriptions, ElDescriptionsItem,
  ElMenu, ElMenuItem, ElDropdown, ElDropdownMenu, ElDropdownItem,
  ElDialog, ElPopover, ElEmpty,
  ElCard, ElTimeline, ElTimelineItem,
  ElConfigProvider,
} from 'element-plus'

export function setupElementPlus(app: App): void {
  for (const [name, comp] of components) {
    app.component(name, comp)
  }
}
```

### 中文 Locale 适配

移除 `app.use(ElementPlus, { locale: zhCn })` 后，需要单独配置中文语言：

```vue
<!-- App.vue -->
<el-config-provider :locale="zhCn">
  <AppLayout />
</el-config-provider>
```

## 效果对比

| 指标 | 优化前 | 优化后 | 改善 |
|------|--------|--------|------|
| 注册组件数 | 200+ | **31** | **-85%** |
| vendor-element JS | 1,015 KB | **603 KB** | **-41%** |
| Build 时间 | 5.50s | **4.97s** | **-10%** |
| Dev 冷启动 | ~8s | **~2s** | **-75%** |
| 页面切换 | 3-5s | **<1s** | **-80%** |

> Dev 模式的改善最为显著：Vite 预构建模块数从 ~1000 降至 ~150，浏览器首次加载的 JS 体积从 ~1.5MB 降至 ~900KB。

## 涉及文件

| 文件 | 变更 | 说明 |
|------|------|------|
| `frontend/src/element-plus.ts` | **新建** | 31 个组件的按需注册模块 |
| `frontend/src/main.ts` | **修改** | 移除 `import ElementPlus` + `app.use(ElementPlus)` |
| `frontend/src/App.vue` | **修改** | 添加 `<el-config-provider :locale="zhCn">` 包裹 |
| `frontend/vite.config.ts` | **修改** | proxy 超时 30s |

## 验证

```bash
cd frontend
npx vue-tsc --noEmit   # TypeScript 类型检查 ✅
npx vite build           # 生产构建 ✅ (4.97s, 603KB vendor-element)
```

## 注意事项

- `ElMessage`、`ElMessageBox` 等命令式 API 仍由各页面直接 `import { ElMessage } from 'element-plus'` — 这些是轻量 JS 模块，不影响性能
- 新增页面使用新的 `<el-*>` 组件时，需同步添加到 `element-plus.ts` 的 import 列表中
- CSS 仍使用全量 `import 'element-plus/dist/index.css'`（单个文件，不构成性能瓶颈）
