## 2. 仪表盘"即将到期"计数修复

### 问题

Dashboard 统计卡片"即将到期"的数字包含了**已过期**的合同。

**根因**：`get_expiring_contracts()` SQL 条件 `julianday(end_date) - julianday("now") <= 30` 无下限，已过期合同（days_left < 0）也被统计。

### 解决方案

在 API 响应中新增 `total_upcoming` 字段（仅计 0~30 天到期），Dashboard 使用该字段：

| 字段 | 含义 | 用途 |
|------|------|------|
| `total` | 全部匹配（含已过期） | ExpiringPanel 预警面板 |
| `total_upcoming` | 仅未过期（days_left ≥ 0） | Dashboard 统计卡片 |

Dashboard 侧边栏列表仍展示全部（已过期 + 即将到期），每项带 `已过期`（红）或 `X天后到期`（橙）标签。
