## 1. 审查目的

本次审查用于确认 PRD、SRS、页面交互说明和系统设计之间是否存在影响测试验收、实施计划或编码启动的冲突，重点覆盖：

- MVP 范围与非目标边界。
- 三角色权限体系与操作边界。
- 合同状态流转（草稿→审批中→已通过→已归档/已作废）。
- 审批链引擎：模板配置、条件匹配、逐级流转、驳回/撤回。
- 字段、校验和数据模型一致性。
- 页面操作与 API 契约映射。
- FTS5 全文检索与到期预警。
- SQLite 持久化与数据库初始化。
- JWT 认证流程与密码安全。
- 5 人并行开发策略与模块解耦。

## 7. 对测试验收方案的输入

后续测试与验收方案应重点覆盖：

- **认证与权限**：登录/注册、JWT 过期处理、三角色权限隔离、审批子角色匹配。
- **合同生命周期**：拟制→保存草稿→提交审批→审批流转→归档的端到端流程。
- **审批链引擎**：模板匹配（条件组合）、逐级审批通过、驳回回到草稿、撤回、无匹配模板拒绝提交。
- **状态机保护**：非法状态转换被后端拒绝（如审批中不可编辑、已归档不可修改）。
- **全文检索**：中文关键词搜索、组合筛选、FTS5 高亮、搜索结果分页。
- **到期预警**：三级分类正确性、紧急程度排序、天数计算准确。
- **审计日志**：关键操作全量记录、管理员筛选查看、不可删除修改。
- **数据持久化**：SQLite 重启后数据保留，首次初始化不覆盖已有数据。
- **安全**：密码 bcrypt 哈希、API Key 不暴露在前端、未认证请求被拦截、审计日志防篡改。
- **前端交互**：表单校验、loading 状态、空状态、二次确认、401 自动跳转登录页。
