本文档定义企业合同管理系统 MVP 的完整技术规格，包括数据库设计、API 契约、前端路由、错误码和业务规则。开发团队以本文档为技术实现依据。

| 层级 | 技术 | 版本要求 |

## 2. 数据库设计

### 2.1 users 表

| 字段 | 类型 | 约束 | 说明 |
|------|------|------|------|
| id | INTEGER | PK, AUTOINCREMENT | 用户ID |
| username | VARCHAR(50) | UNIQUE, NOT NULL | 用户名 |

| role | VARCHAR(20) | NOT NULL | 角色(handler/approver/admin) |

| is_active | BOOLEAN | DEFAULT TRUE | 是否启用 |
| created_at | DATETIME | NOT NULL | 创建时间 |

| 字段 | 类型 | 约束 | 说明 |
|------|------|------|------|
| id | INTEGER | PK, AUTOINCREMENT | 合同ID |

| created_at | DATETIME | NOT NULL | 创建时间 |
| updated_at | DATETIME | NOT NULL | 更新时间 |

### 2.3 approval_chain_templates 表

| 字段 | 类型 | 约束 | 说明 |
|------|------|------|------|
| id | INTEGER | PK, AUTOINCREMENT | 模板ID |
| name | VARCHAR(100) | NOT NULL | 模板名称 |
| conditions | TEXT | NOT NULL | 触发条件(JSON) |
| steps | TEXT | NOT NULL | 审批步骤(JSON数组) |
| priority | INTEGER | DEFAULT 0 | 优先级(数值越大越优先) |
| is_active | BOOLEAN | DEFAULT TRUE | 是否启用 |
| created_at | DATETIME | NOT NULL | 创建时间 |

**conditions JSON 格式：**
```json
{
  "contract_type": ["purchase", "sales"],
  "amount_min": 10000,
  "amount_max": 500000
}
```
所有条件为 AND 关系，空字段表示不限制。

**steps JSON 格式：**
```json
[
  {"name": "部门审批", "role": "dept_manager", "timeout_hours": 24},
  {"name": "法务审核", "role": "legal", "timeout_hours": 48},
  {"name": "财务审批", "role": "finance_director", "timeout_hours": 24}
]
```

### 2.7 FTS5 全文索引

```sql
CREATE VIRTUAL TABLE contracts_fts USING fts5(
  title,
  content,
  content='contracts',
  content_rowid='id'
);

-- 触发器：自动同步
CREATE TRIGGER contracts_ai AFTER INSERT ON contracts BEGIN
  INSERT INTO contracts_fts(rowid, title, content) VALUES (new.id, new.title, new.content);
END;

CREATE TRIGGER contracts_ad AFTER DELETE ON contracts BEGIN
  INSERT INTO contracts_fts(contracts_fts, rowid, title, content) VALUES('delete', old.id, old.title, old.content);
END;

CREATE TRIGGER contracts_au AFTER UPDATE ON contracts BEGIN
  INSERT INTO contracts_fts(contracts_fts, rowid, title, content) VALUES('delete', old.id, old.title, old.content);
  INSERT INTO contracts_fts(rowid, title, content) VALUES (new.id, new.title, new.content);
END;
```

## 3. API 契约

### 3.1 通用约定

- 基础路径：`/api/v1`
- 认证方式：`Authorization: Bearer <token>`
- 请求格式：`application/json`
- 响应格式：`application/json`
- 成功响应：`{"data": ..., "message": "success"}`
- 错误响应：`{"detail": "错误描述", "error_code": "ERROR_CODE"}`

### 3.2 认证模块

**POST /api/v1/auth/login**
- 请求：`{"username": "str", "password": "str"}`
- 响应：`{"data": {"access_token": "str", "token_type": "bearer", "user": {...}}, "message": "success"}`
- 错误：401 `{"detail": "用户名或密码错误"}`

**POST /api/v1/auth/register**
- 权限：admin
- 请求：`{"username": "str", "password": "str", "role": "str", "approver_role": "str|null", "department": "str", "display_name": "str"}`
- 响应：`{"data": {...user}, "message": "success"}`
- 错误：400 `{"detail": "用户名已存在"}`

**GET /api/v1/auth/me**
- 响应：`{"data": {...user}, "message": "success"}`

### 3.3 合同模块

**GET /api/v1/contracts**
- 参数：`?status=&type=&page=1&page_size=10`
- 响应：`{"data": {"items": [...], "total": 100, "page": 1, "page_size": 10}, "message": "success"}`

**POST /api/v1/contracts**
- 权限：handler
- 请求：`{"title": "str", "content": "str", "contract_type": "str", "amount": float|null, "party_a": "str", "party_b": "str", "start_date": "date", "end_date": "date"}`
- 响应：`{"data": {...contract}, "message": "success"}`

**GET /api/v1/contracts/{id}**
- 响应：`{"data": {...contract, approval_history: [...]}, "message": "success"}`

**PUT /api/v1/contracts/{id}**
- 权限：handler (仅 draft/voided 状态)
- 请求：同上创建
- 错误：409 `{"detail": "当前状态不允许编辑"}`

**POST /api/v1/contracts/{id}/submit**
- 权限：handler
- 触发审批链匹配
- 错误：400 `{"detail": "未找到匹配的审批链模板"}`

**POST /api/v1/contracts/{id}/withdraw**
- 权限：handler (仅 pending_approval 状态)
- 审批实例状态变为 withdrawn

**POST /api/v1/contracts/{id}/archive**
- 权限：handler (仅 approved 状态)

**POST /api/v1/contracts/{id}/void**
- 权限：handler (仅 draft 状态)

### 3.4 审批模块

**GET /api/v1/approvals/pending**
- 权限：approver
- 返回当前用户作为当前步骤审批人的待审批合同列表

**POST /api/v1/approvals/{instance_id}/approve**
- 权限：当前步骤审批人
- 请求：`{"comment": "str"}`
- 全部步骤通过后合同状态自动变为 approved

**POST /api/v1/approvals/{instance_id}/reject**
- 权限：当前步骤审批人
- 请求：`{"comment": "str"}`
- 合同状态回到 draft，记录驳回步骤和理由

### 3.5 审批链配置模块

**GET /api/v1/approval-chains**
- 权限：admin
- 返回所有审批链模板列表

**POST /api/v1/approval-chains**
- 权限：admin
- 请求：`{"name": "str", "conditions": {...}, "steps": [...], "priority": int}`

**PUT /api/v1/approval-chains/{id}**
- 权限：admin

**DELETE /api/v1/approval-chains/{id}**
- 权限：admin
- 仅当无进行中的审批实例引用时可删除

### 3.6 搜索模块

**GET /api/v1/search**
- 参数：`?q=关键词&type=&status=&date_from=&date_to=&page=1&page_size=10`
- 响应：`{"data": {"items": [...], "total": N, "highlights": {"contract_id": ["matched text..."]}}, "message": "success"}`

### 3.7 预警模块

**GET /api/v1/contracts/expiring**
- 参数：`?days=30`
- 返回 end_date 在指定天数内或已过期的合同列表
- 按紧急程度排序

### 3.8 模板模块

**GET /api/v1/templates**
- 返回可用合同模板列表

**POST /api/v1/templates**
- 权限：handler/admin
- 请求：`{"name": "str", "title_template": "str", "content_template": "str", "contract_type": "str"}`

### 3.9 审计日志模块

**GET /api/v1/audit-logs**
- 权限：admin
- 参数：`?user_id=&action=&page=1&page_size=20`

### 3.10 用户管理模块

**GET /api/v1/users**
- 权限：admin
- 返回用户列表

**PUT /api/v1/users/{id}**
- 权限：admin
- 修改角色、部门、启用状态

## 4. 业务规则

### 4.1 合同状态机

```
draft → (submit) → pending_approval → (all_approved) → approved → (archive) → archived
                         ↓ reject                         
                      (back to draft)                     
                         ↓ withdraw
                      (back to draft)
draft → (void) → voided
```

### 4.2 审批链匹配规则

1. 过滤 is_active = TRUE 的模板。
2. 按 priority DESC, created_at ASC 排序。
3. 逐个检查条件：
   - contract_type：合同类型在列表中或条件为空。
   - amount_min：合同金额 >= 条件值或条件为空。
   - amount_max：合同金额 <= 条件值或条件为空。
4. 返回第一个完全匹配的模板。
5. 无匹配时拒绝提交。

### 4.3 审批权限规则

- 审批实例当前步骤的 role 与当前用户的 approver_role 匹配时，该用户有权审批。
- 同一用户不能既是经办人又是审批人（同一合同）。

## 5. 错误码

| HTTP状态码 | error_code | 说明 |
|------------|------------|------|
| 400 | VALIDATION_ERROR | 请求参数校验失败 |
| 400 | NO_APPROVAL_CHAIN | 未找到匹配的审批链 |
| 401 | UNAUTHORIZED | 未认证或 token 过期 |
| 403 | FORBIDDEN | 权限不足 |
| 404 | NOT_FOUND | 资源不存在 |
| 409 | STATUS_CONFLICT | 当前状态不允许此操作 |
| 409 | DUPLICATE_USERNAME | 用户名已存在 |
| 500 | INTERNAL_ERROR | 服务器内部错误 |
