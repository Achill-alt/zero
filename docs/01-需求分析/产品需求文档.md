## 1. 产品概述

企业合同管理系统是为企业内部打造的一站式合同管理平台，覆盖合同从拟制到归档的完整生命周期。核心提供动态审批链引擎、全文检索和到期预警三大能力。

**目标用户：** 企业员工（经办人）、部门审批人、系统管理员

**产品形态：** Web 应用（桌面浏览器）

## 2. 功能需求

### FR-01：用户认证与角色管理

**FR-01.1 注册**
- 管理员可注册新用户，指定用户名、密码、角色和部门。
- MVP 不开放自助注册，由管理员统一管理账号。

**FR-01.2 登录**
- 用户使用用户名+密码登录，系统返回 JWT token。
- Token 有效期 24 小时，过期后需重新登录。
- 前端存储 token 于 localStorage，每次请求通过 Authorization header 携带。

**FR-01.3 角色管理**
- 系统预设三种角色：经办人、审批人、管理员。
- 审批人角色进一步细分：dept_manager、legal、finance_director、ceo。
- 管理员可为用户分配和修改角色。

**FR-01.4 权限控制**
- 后端通过 JWT 中间件验证身份。
- 通过角色中间件控制接口访问权限。
- 前端根据角色展示不同菜单和操作按钮。

### FR-02：合同管理

**FR-02.1 合同拟制**
- 经办人可选择空白合同或预设模板创建合同。
- 填写合同标题、内容（富文本）、金额、类型、甲方/乙方、起止日期。
- 草稿可随时保存，不触发生效。

**FR-02.2 合同编辑**
- 草稿和退回状态的合同可编辑。
- 审批中、已归档的合同不可编辑。
- 合同作废为不可逆操作。

**FR-02.3 合同列表**
- 展示当前用户相关的合同列表。
- 支持按状态、类型筛选。
- 支持按创建时间排序。
- 分页展示，每页 10 条。

**FR-02.4 合同详情**
- 展示合同全部字段、内容。
- 展示审批历史和当前审批进度。
- 根据状态提供不同操作按钮。

### FR-03：审批链引擎

**FR-03.1 审批链模板配置**
- 管理员创建审批链模板：名称、触发条件（JSON）、审批步骤（JSON）。
- 触发条件支持：contract_type（合同类型）、amount_min/amount_max（金额范围）。
- 审批步骤：有序列表，每步包含 name、role、timeout_hours。
- 模板可启用/禁用，禁用的模板不参与匹配。

**FR-03.2 审批链匹配**
- 合同提交时，系统按模板创建时间顺序匹配第一个满足条件且启用的模板。
- 无匹配模板时，合同不允许提交，提示"未找到匹配的审批链"。

**FR-03.3 审批流转**
- 匹配成功后生成审批实例，从 step_0 开始。
- 当前步骤审批人通过后自动进入下一步。
- 所有步骤通过后合同进入待归档状态。
- 任意步骤驳回后合同回到草稿状态，记录驳回理由。
- 经办人可随时撤回审批中的合同。

**FR-03.4 审批操作**
- 审批人查看待审批合同列表。
- 审批人通过或驳回合同，驳回必须填写理由。
- 审批操作记录审计日志。

### FR-04：全文检索

**FR-04.1 搜索功能**
- 基于 SQLite FTS5 实现，对合同标题和内容建立全文索引。
- 支持中文关键词搜索。
- 支持按合同类型、日期范围、状态组合筛选。
- 搜索结果按相关度排序，高亮匹配关键词。

**FR-04.2 搜索入口**
- 全局搜索栏位于顶部导航。
- 合同列表页提供搜索和筛选入口。

### FR-05：到期预警

**FR-05.1 预警计算**
- 系统根据合同结束日期自动计算到期状态。
- 分类：已过期（end_date < today）、30天内到期、正常。

**FR-05.2 预警面板**
- 独立页面，展示到期和已过期合同列表。
- 按紧急程度排序：已过期 > 7天内到期 > 30天内到期。
- 每项显示：合同标题、结束日期、距今天数、当前状态。
- 点击跳转到合同详情。

### FR-06：合同模板

**FR-06.1 模板管理**
- 管理员和经办人可创建合同模板。
- 模板包含预设的标题和内容，拟制时自动填充。

**FR-06.2 模板使用**
- 经办人拟制合同时可选择模板。
- 选择模板后自动填充标题和内容，字段仍可修改。

### FR-07：审计日志

**FR-07.1 日志记录**
- 自动记录关键操作：登录、合同创建/编辑/提交/审批/驳回/撤回/归档/作废。
- 日志字段：user_id、action、target_type、target_id、detail（JSON）、created_at。

**FR-07.2 日志查看**
- 管理员专用页面。
- 支持按用户、操作类型、时间范围筛选。
- 按时间倒序排列。

## 3. 非功能需求

### NFR-01：性能
- 页面首次加载时间不超过 3 秒。
- 搜索响应时间不超过 1 秒。
- API 响应时间（除搜索外）不超过 500ms。

### NFR-02：安全
- 密码使用 bcrypt 哈希存储。
- JWT 密钥通过环境变量配置。
- API 所有端点（除登录/注册外）需要认证。
- 审计日志不可删除或修改。

### NFR-03：可用性
- 桌面浏览器适配（Chrome、Edge、Firefox 最新版）。
- 表单校验实时反馈。
- 操作成功/失败有明确提示。

### NFR-04：数据持久化
- 使用 SQLite 数据库文件持久化。
- 应用重启后数据保留。
- 数据库文件首次运行时自动创建和初始化。
