## 1. 设计目标

- 前后端分离，通过 RESTful API 通信。
- 后端分层架构（Router → Service → Repository → Model），清晰职责边界。
- 前端组件化开发，Vue 3 Composition API。
- 5人可并行开发，模块间低耦合。
- 审批链引擎作为独立可扩展模块。

## 3. 后端分层架构

```
┌─────────────────────────────────────────┐
│  api/  (Router 层)                       │
│  - 路由注册、参数解析、HTTP 响应           │
│  - 依赖注入（get_db, get_current_user）    │
└──────────────┬──────────────────────────┘
               │ 调用
┌──────────────▼──────────────────────────┐
│  services/  (Service 层)                 │
│  - 业务逻辑编排                           │
│  - 事务管理                               │
│  - 审批链匹配、状态机校验                   │
└──────────────┬──────────────────────────┘
               │ 调用
┌──────────────▼──────────────────────────┐
│  repositories/  (Repository 层)          │
│  - 数据访问封装                           │
│  - SQLAlchemy 查询                        │
│  - FTS5 搜索查询                          │
└──────────────┬──────────────────────────┘
               │ 操作
┌──────────────▼──────────────────────────┐
│  models/  (Model 层)                     │
│  - SQLAlchemy ORM 模型定义                │
│  - 表关系和约束                           │
└─────────────────────────────────────────┘
```

**依赖方向：** api → services → repositories → models（单向依赖，上层不可被下层引用）

## 4. 模块解耦设计

### 4.1 5人并行开发策略

| 人员 | 模块 | 依赖 | 独立开发策略 |
|------|------|------|-------------|
| P1 | 认证+用户 | 无 | 先交付 User 模型 + JWT 中间件，其他人依赖它 |
| P2 | 合同CRUD+模板 | P1 (User模型) | P1 交付 User 模型后可开始，Contract 模型独立 |
| P3 | 审批链引擎 | P1, P2 | P1/P2 模型交付后开始，但可先写 Service 层逻辑 |
| P4 | 全文检索+归档 | P1, P2 | 可先写 FTS5 初始化脚本，等 P2 模型交付后集成 |
| P5 | 前端页面 | 后端API | 先搭建 Vue 项目骨架、路由、布局，用 mock 数据开发页面 |

### 4.2 集成顺序

```
Step 1: P1 交付 User 模型 + JWT 中间件（最早）
Step 2: P2 交付 Contract 模型 + CRUD API
Step 3: P3 审批链引擎对接 P1+P2
Step 4: P4 搜索/归档对接 P1+P2
Step 5: P5 前端逐页对接后端 API
```

## 5. 关键设计决策

### 5.1 审批链匹配引擎

- 合同提交时，`approval_chain_service.match(contract)` 按优先级匹配模板。
- 匹配逻辑独立封装，方便后续扩展更多条件类型。
- 审批实例创建时深拷贝模板步骤为 `step_results` 初始数组。

### 5.2 FTS5 全文索引

- 在 `database.py` 中通过 SQLAlchemy 事件监听创建 FTS5 虚拟表和触发器。
- 搜索时使用 `MATCH` 查询 + `snippet()` 函数生成高亮片段。
- 中文分词依赖 FTS5 内置的 Unicode 分词器（对中文按字符级处理）。

### 5.3 JWT 认证流程

```
1. 用户 POST /auth/login {username, password}
2. 服务端验证密码，生成 JWT (sub=user_id, exp=24h)
3. 前端存 token 到 localStorage
4. Axios 拦截器自动在请求头添加 Authorization
5. 后端 auth 中间件解码 JWT，注入 current_user
6. role 中间件检查角色权限
```

### 5.4 前端组件复用

- `ContractForm.vue` 同时用于拟制页和编辑页。
- `AppLayout.vue` 统一全局布局框架。
- `ApprovalTimeline.vue` 在合同详情中复用。
- `StatCard.vue` 在工作台和预警面板复用。

### 5.5 错误处理策略

- Service 层抛出业务异常（`HTTPException`）。
- Router 层捕获并转换为统一错误响应格式。
- 前端 Axios 响应拦截器统一处理 401 跳转和错误提示。

## 6. 数据库初始化

- 应用启动时 `database.py` 自动创建所有表和 FTS5 索引。
- `seed.py` 脚本初始化管理员账号和默认审批链模板。
- 首次启动：`python -m app.seed` 或集成到 `main.py` 的 startup 事件。
