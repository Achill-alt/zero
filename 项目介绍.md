## 1. 项目概述

### 1.1 项目定位

**企业合同管理系统**（Enterprise Contract Management System）是一个覆盖合同全生命周期的数字化管理平台。系统支持合同拟制、多级审批流转、全文检索、模板管理、到期预警、审计追溯和数据导出，适用于中小企业的合同管理场景。项目采用前后端分离架构，遵循 **docs-example 7 阶段开发流程**，从需求分析到复盘沉淀全链条文档化。

### 1.2 核心价值

| 维度 | 价值 |
|------|------|
| **效率提升** | 合同拟制支持富文本编辑 + 模板快速创建，告别 Word 排版 |
| **流程规范** | 多级审批引擎，优先级匹配 + JSON 条件驱动，审批链灵活配置 |
| **智能检索** | jieba 中文分词 + SQLite FTS5 全文搜索，秒级定位合同 |
| **风险预警** | 30 天内到期合同自动识别，避免合同逾期 |
| **数据安全** | JWT 认证 + bcrypt 密码哈希 + 角色权限控制 + 审计全追溯 |
| **零配置部署** | Docker Compose 一键启动，跨平台兼容 Windows/Linux/macOS |
| **质量保障** | 44 pytest 单元测试 + 36 Playwright 浏览器 E2E，TypeScript 类型安全 |

### 1.3 适用场景

- 中小企业合同全生命周期管理
- 毕业设计/课程答辩项目（完整的 7 阶段文档体系）
- 全栈开发学习参考（FastAPI + Vue 3 + TypeScript + Playwright）

## 3. 功能详解

### 3.1 认证与权限（Auth & RBAC）

| 功能点 | 实现 |
|--------|------|
| 用户注册 | 用户名/密码/显示名/部门/角色，密码 bcrypt 哈希存储 |
| JWT 登录 | Token 过期机制，localStorage 持久化 |
| 三级角色 | `admin`（管理员）、`handler`（经办人）、`approver`（审批人） |
| 审批子角色 | `dept_manager`（部门负责人）、`legal`（法务）、`finance_director`（财务总监）、`ceo`（总经理） |
| 路由守卫 | 未登录重定向 → `/login`；角色不匹配 → 403 |
| 权限中间件 | `require_role` 依赖注入，越权访问返回 403 |
| 用户管理 | 管理员可新增用户、启用/禁用账户 |

**API 端点：**
- `POST /api/v1/auth/login` — 登录
- `POST /api/v1/auth/register` — 注册
- `GET /api/v1/auth/me` — 获取当前用户信息

### 3.2 合同管理（Contract CRUD）

| 功能点 | 实现 |
|--------|------|
| 拟制合同 | 自由创建 + 从模板快速创建，表单校验 |
| 富文本编辑器 | **TipTap v3** — 加粗 / 斜体 / H2 / H3 / 无序列表 / 有序列表 / 引用 / 撤销 / 重做 |
| 编辑合同 | 仅草稿状态可编辑；编辑器回显已有 HTML 内容 |
| 删除合同 | 仅草稿状态可删除 |
| 合同详情 | DOMPurify 消毒后 `v-html` 渲染，XSS 安全 |
| **5 状态机** | `draft` → `pending_approval` → `approved` → `archived` / `voided` |
| 状态转换 | 提交审批 / 作废 / 驳回 / 撤回 / 审批通过 / 归档 |
| 列表筛选 | 按状态 / 类型 / 分页 / 排序 |

**合同字段：**
- 标题（200 字内）、内容（富文本 HTML）、合同类型（采购/销售/服务/租赁/其他）
- 金额（Decimal 15,2）、甲方、乙方
- 开始日期、结束日期、状态、经办人

**API 端点：**
- `GET /api/v1/contracts` — 分页列表（?status=&type=&page=&page_size=）
- `POST /api/v1/contracts` — 创建合同
- `GET /api/v1/contracts/{id}` — 合同详情
- `PUT /api/v1/contracts/{id}` — 编辑合同
- `DELETE /api/v1/contracts/{id}` — 删除合同
- `POST /api/v1/contracts/{id}/submit` — 提交审批
- `POST /api/v1/contracts/{id}/void` — 作废合同

### 3.3 审批引擎（Approval Engine）

这是系统的核心模块，实现了优先级匹配 + JSON 条件驱动的多级审批流转。

| 功能点 | 实现 |
|--------|------|
| **审批链模板** | CRUD 管理，JSON 条件 + JSON 步骤 |
| **条件匹配** | 合同类型 × 金额范围 × 部门，AND 逻辑组合 |
| **优先级排序** | 数值越大越优先，首次匹配即命中 |
| **多级流转** | 步骤顺序执行（step_0 → step_1 → … → 完成） |
| **审批操作** | 通过（前进到下一步）/ 驳回（合同退回草稿） |
| **撤回操作** | 经办人可撤回，合同退回草稿 |
| **待审列表** | 按当前步骤角色筛选，仅显示当前用户可审批的合同 |
| **内置模板** | "标准四级审批"（部门→法务→财务→总经理）、"小额合同简易审批"（金额 < 5 万） |

**审批链模板示例：**

```json
// conditions：触发条件
{ "amount_min": 50000, "contract_type": ["purchase", "sales"] }

// steps：审批步骤
[
  { "name": "部门审批", "role": "dept_manager", "timeout_hours": 24 },
  { "name": "法务审核", "role": "legal", "timeout_hours": 48 },
  { "name": "财务审批", "role": "finance_director", "timeout_hours": 24 },
  { "name": "总经理批准", "role": "ceo", "timeout_hours": 72 }
]
```

**状态机流转：**

```
draft ──提交审批──→ pending_approval ──全部通过──→ approved ──归档──→ archived
  │                      │                                          
  └──作废──→ voided       ├──驳回──→ draft
                          └──撤回──→ draft
```

**API 端点：**
- `GET /api/v1/approvals/pending` — 待审批列表（按角色筛选）
- `POST /api/v1/approvals/{instance_id}/approve` — 通过
- `POST /api/v1/approvals/{instance_id}/reject` — 驳回
- `POST /api/v1/approvals/{instance_id}/withdraw` — 撤回
- `GET/POST/PUT/DELETE /api/v1/approval-chains` — 审批链模板 CRUD

### 3.4 全文检索（Full-text Search）

| 功能点 | 实现 |
|--------|------|
| **英文搜索** | SQLite FTS5 + unicode61 tokenizer，高亮片段 |
| **中文搜索** | jieba 分词 + 应用层 token 拼接 + FTS5 MATCH 查询 |
| **高亮片段** | `snippet()` 函数返回匹配上下文 |
| **组合筛选** | 搜索词 + 状态 + 合同类型，三者可组合使用 |
| **搜索范围** | 合同标题 + 合同内容 + 甲方 + 乙方 |

**技术细节：**
- FTS5 外部内容表（`contracts_fts`）+ 触发器自动同步
- jieba 精确模式分词 → 空格拼接 → FTS5 MATCH 查询
- LIKE fallback 兜底（分词后无有效 token 时）

**API 端点：**
- `GET /api/v1/search?q=关键词&status=&type=&page=&page_size=`

| `attachments` | 附件 | contract_id, original_filename, stored_filename, file_size, mime_type |
| `notifications` | 通知 | user_id, contract_id, type, message, is_read |
| `audit_logs` | 审计日志 | user_id, action, target_type, target_id, ip_address, details |
| `contracts_fts` | FTS5 虚拟表 | title, content, party_a, party_b（外部内容表） |

### 5.2 数据关系

```
users 1──N contracts        (经办人)
users 1──N approval_records (审批人)
users 1──N notifications    (通知接收人)

contracts 1──1 approval_instances  (当前审批)
contracts 1──N attachments         (附件)
contracts 1──N notifications       (通知)

approval_chain_templates 1──N approval_instances
approval_instances 1──N approval_records

contracts_fts ←──触发器同步── contracts (标题、内容、甲方、乙方)
```

## 附录

### A. 技术决策记录

| 决策 | 选择 | 理由 |
|------|------|------|
| 数据库 | SQLite（非 PostgreSQL） | MVP 零配置，FTS5 满足全文检索 |
| 密码加密 | bcrypt 直接使用（非 passlib） | 避免 bcrypt 5.x 兼容性问题 |

| UI 组件库 | Element Plus（非 Ant Design Vue） | 中文文档完善，社区活跃 |
| 富文本编辑器 | TipTap（非 Quill） | Vue 3 一等公民，TypeScript 原生 |
| 中文分词 | jieba（非 pkuseg） | 轻量、成熟、中文社区广泛使用 |
| PDF 引擎 | WeasyPrint（非 ReportLab） | CSS 渲染、维护活跃、Jinja2 模板 |
| 前端语言 | TypeScript（v1.0 为 JavaScript） | 渐进迁移，类型安全 |
